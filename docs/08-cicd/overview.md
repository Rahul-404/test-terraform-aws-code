# Overview

## Purpose

This document provides an overview of the Continuous Integration and Continuous Delivery (CI/CD) strategy used throughout the platform.

CI/CD enables the platform to deliver infrastructure, applications, and machine learning systems in a reliable, repeatable, and automated manner.

The objectives of the CI/CD platform are:

* Reduce manual operations
* Improve deployment consistency
* Increase delivery velocity
* Improve system reliability
* Enable safe releases
* Support platform scalability

CI/CD is a foundational capability of the platform.

---

# What CI/CD Means in This Platform

The platform treats infrastructure, applications, and machine learning assets as deployable artifacts.

Examples:

```text id="v5k8nt"

Applications

Container Images

Machine Learning Models

Configuration
```

All changes follow automated delivery workflows.

The platform CI/CD system is designed to:

### Automate Repetitive Tasks

CI/CD pipelines follow several security principles.

---

## Temporary Credentials

Long-lived credentials should be avoided.

Preferred:
